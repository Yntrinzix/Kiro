# Parked Idea: Composable/Contractual Agents

Date: 2026-05-11

## Concept
Assemble niche agents on-demand from three blocks:
- Purpose (what it does, when to use it)
- Skills (technical knowledge, pulled from existing skill files)
- Behaviour (adversarial, optimistic, pragmatic, cautious)

## Example
Fullstack engineer = purpose(end-to-end features) + skills(dotnet, grpc, nextjs, react) + behaviour(pragmatic)

## Why Parked
Current roster of 12 agents covers the problem space. Ad-hoc subagent prompts handle edge cases fine. This solves a problem that would appear if:
- Working across many different stacks regularly
- Needing dozens of niche agent variations
- Building a multi-tenant AI platform
- Roster grows past ~20 agents

## Revisit When
- New client with unfamiliar stack where writing full agent guides feels heavy
- Team grows and needs consistent agent assembly
- Agent roster maintenance becomes a burden
