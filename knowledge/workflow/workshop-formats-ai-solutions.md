# Workshop Formats for AI Solutions Engagements

## 1. Problem Definition Workshop
- Everyone writes their definition of the problem independently
- Group synthesises into one shared definition
- Define constraints (budget, timeline, tech, data availability)
- Define success criteria (how do we know it worked?)
- Identify unknowns and risks
- Output: one-page problem statement everyone agrees on

## 2. How Might We (diverge then converge)
- Start with the agreed problem statement
- Everyone generates "How might we..." questions (no solutions yet)
- Cluster similar ones
- Dot-vote on highest impact
- THEN brainstorm solutions for the top 3
- Output: prioritised solution directions

## 3. Assumption Mapping
- List every assumption the proposed solution relies on
- Plot on a 2x2: (known/unknown) vs (risky/safe)
- Unknown + risky = must validate before building
- Known + safe = just do it
- Output: what to prototype/test first

## 4. User Journey Mapping (AI-specific)
- Map the current workflow step by step (manual, painful)
- For each step: could AI help? What type? (classification, generation, extraction, decision)
- Mark: full automation vs human-in-the-loop vs no AI needed
- Identify data requirements for each AI step
- Output: where AI fits and where it doesn't

## 5. Feasibility/Effort Matrix
- List all proposed features/capabilities
- Score each: feasibility (data available? tech proven?) vs impact (revenue, time saved, error reduction)
- High feasibility + high impact = build first
- Low feasibility + high impact = spike/research
- Output: phased roadmap

## 6. Pre-mortem
- "It's 6 months from now. The project failed. Why?"
- Everyone writes failure scenarios independently
- Share and cluster
- For each cluster: what would we do NOW to prevent it?
- Output: risk mitigation plan baked into the project from day one

## Recommended Sequence for New Client Engagement

```
Workshop 1: Problem Definition (#1)
Workshop 2: User Journey + Where AI Fits (#4)
Workshop 3: Assumption Mapping (#3)
Workshop 4: Feasibility Matrix (#5) → becomes the proposal
```
